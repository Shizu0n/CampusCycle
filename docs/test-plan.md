## Affected Pages/Routes
- `/` (Landing, desktop) — hero, impact counter (real stats + simulated floor), showcase with category filters, CTAs. Verify counter shows plausible non-zero numbers with seed data.
- `/feed` — listing grid, category filter, search `?q=`, donation filter, pagination (12/page). Verify offline: last feed visible (NetworkFirst).
- `/listings/:id` — minimal detail page reached by card click.
- `/new` (NewListing form) — title/description/category/price-or-donation/image URL (optional → category placeholder). Client-side zod validation BEFORE queueing (Issue 5B).
- `/mine` (MyListings) — merged serverItems + queuePendingItems by client UUID; sync badges (pending / failed / published).
- `/login`, `/register` — JWT stage 2; register ALWAYS creates a new user (never claims anonymous stub).
- API: `/api/listings` (CRUD+filters), `/api/auth/*`, `/api/stats`, `/api/health`.

## Key Interactions to Verify
- Create listing online → appears in feed and /mine immediately.
- Create listing OFFLINE (airplane mode) → "aguardando sincronização" badge → reconnect → badge flips to "publicado", item present server-side, NO duplicate.
- Retry idempotency: force flaky network during sync — same client UUID reused, exactly one listing created.
- Login identity swap → API runtime cache purged (Issue 3A) → /mine offline never shows pre-login identity's cached list.
- PATCH: mark item as sold → status changes, impact counter updates.
- Delete own listing; attempt PATCH/DELETE on someone else's listing → 403 in error envelope.
- Rate limit: >20 writes/15min from one IP → 429 in envelope; UI surfaces retry state (queue: pending + backoff).
- Cold start: first request against sleeping Render/Neon → "Servidor gratuito acordando (~1 min)…" message with auto-retry; cached feed visible meanwhile.

## Edge Cases
- Queue state machine (all must be visible, never silent): network fail → pending; 4xx validation → failed "toque para revisar" (reopens filled form); 401 expired JWT → pending + re-login prompt, syncs after; 429 → pending + backoff; 5xx/5th retry → failed with reason.
- P2002 duplicate id: same owner → idempotent 200; DIFFERENT owner → 409 (Issue 4A).
- Double-click submit: button disabled in-flight; UUID regenerated only after confirmed success.
- App killed with pending queue items → reopen → retry on load fires.
- Empty states: zero listings in a filtered category; zero "my listings"; stats with empty DB (simulated floor keeps numbers non-zero).
- Boundary inputs: max-length title/description (zod max lengths); price 0 vs null (donation); invalid image URL → category placeholder.
- Installed PWA: close app on an INNER route, reopen from home icon while offline → app opens (navigateFallback).

## Critical Paths
- Airplane-mode announce (×3 consecutive clean runs before recording, on the pre-req-0 demo device, in Chrome): login → airplane mode → fill form → pending badge → reconnect → published badge → item in /mine and feed. Includes 3A purge and inner-route-offline-reopen checks.
- Register → login → create → mark sold → delete, all against production (Render+Neon+Vercel), after warm-up hit on /api/stats.
- Install PWA from Chrome → standalone launch → full announce flow inside installed app.
